<readonly> events :String

Events that MultiRoute will emit from the view that is performing the action.
Properties:
Name Type Default Description
CHANGE String MultiRouteChange When the route has been changed in the url and is starting its loading and resolving. This will send back the currentPath and the previousPath
BEFORE_CHANGE String MultiRouteBeforeChange Before the change happens. This is just before a route gets activated to be set for change. This will send back the currentPath and the nextPath
ROUTE_COMPLETE String MultiRouteQueueManagerRouteReady As each route completes this is dispatched from the view it completed in.
PARAMS_CHANGE String MultiRouteParamsChange When the search params in the url change but the views have not, this will dispatch so that the views can fire their VIEW_READY event after these have been applied.
VIEW_READY String MultiRouteViewReady After the view is ready, as in loaded, resolved, and transition is completed.

<readonly> routeStatus :Number

The status phases that each path passed to the multiRoute will go through before the next path is able to begin it's phase cycle.
Properties:
Name Type Default Description
WAITING Number 1 Status of a path that is waiting in queue.
LOADING Number 2 The status of path that is in loading its dependencies.
LOADED Number 3 The status of a path that has loaded it's dependencies.
ENTERING Number 4 The status of a path that is currently transitioning into view.
READY Number 5 The status of a path that has finished all transitions.
